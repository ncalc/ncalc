# Security Policy

## Supported Versions

Security updates are provided only for the latest release available on GitHub. Older releases and all previous versions are no longer supported.

| Version               | Supported |
| --------------------- | --------- |
| Latest GitHub Release | ✅         |
| Older Releases        | ❌         |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it by email to:

**[gustavomauriciodebarros@gmail.com](mailto:gustavomauriciodebarros@gmail.com)**

When reporting a vulnerability, please include:

* A description of the issue
* Steps to reproduce the vulnerability
* Potential impact
* Any proof-of-concept code or screenshots, if applicable

After receiving a report, we will acknowledge receipt as soon as reasonably possible and investigate the issue. We will keep the reporter informed of our progress when appropriate.

Valid vulnerabilities will be addressed in a future release. We kindly ask that you do not publicly disclose security vulnerabilities until a fix has been released and users have had a reasonable opportunity to update.

Reports determined not to be security vulnerabilities, duplicate reports, or issues that cannot be reproduced may be closed without further action.
